loupe opens any git diff in an Azure DevOps-style review UI on localhost. Leave inline comments — then compile them into a structured review prompt your coding agent can act on.

$ git clone https://github.com/codywilliamson/loupe && cd loupe $ bun install && bun link $ cd ~/your/repo && loupe ✻ loupe v0.6.0 — reviewing working tree (12 files changed) http://localhost:49507 (ctrl+c to stop)

Made for the review, not the merge button

✻Comment on any line

Single lines, drag-selected ranges, both sides of a split view, even removed lines and whole files. Tag each one nit, issue, question, or praise.

✻Compile a review prompt

One click turns every comment — with surrounding diff context — into structured markdown ready to paste into Claude, or any LLM that writes code.

✻Diffs worth reading

Unified or side-by-side, syntax highlighting, word-level change marks inside modified lines, sticky file headers, markdown previews.

✻Keyboard-driven

Walk files with j/k, mark them viewed with v, flip views, compile — without touching the mouse. ? shows the full map.

✻Live while agents work

The diff re-runs on every refresh, untracked files show up as additions, and a viewed-progress bar tracks what you've actually read.

✻Four themes, one of them ours

Light, dark, and two Claude-inspired warm variants — ivory paper and soft charcoal with terracotta accents.

✻

Three steps, no ceremony

Run loupe in a repo

Working tree, staged changes, a branch comparison, or any commit range — it parses the diff and opens your browser.

Review like a PR

Read, comment, tag, and check files off. Everything saves to a local .review file, auto-ignored by git.

Brief your agent

Compile Review Prompt assembles your comments with code context. Paste it into your agent and let it do the rework.

✻

The export is the point

Review tools collect comments; loupe compiles them. Each comment is anchored to its lines, wrapped in diff context, labeled with your intent, and ordered by file — so the model fixes exactly what you flagged.
